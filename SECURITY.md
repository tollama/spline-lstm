# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

We take the security of spline-lstm seriously. If you discover a security vulnerability, please report it responsibly.

### How to Report

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via one of the following methods:

1. **GitHub Security Advisory** (preferred): Use the [Security Advisories](https://github.com/ychoi-atop/spline-lstm/security/advisories) feature to privately report a vulnerability.

2. **Email**: Send details to the maintainer directly.

### What to Include

Please include the following information:

- Type of vulnerability (e.g., injection, XSS, data exposure)
- Full path of source file(s) related to the manifestation
- Steps to reproduce the issue
- Proof-of-concept or exploit code (if available)
- Impact assessment

### Response Timeline

- **Initial response**: Within 48 hours
- **Triage and confirmation**: Within 5 business days
- **Fix development**: Depends on severity and complexity
- **Disclosure**: After fix is released

### Disclosure Policy

We follow responsible disclosure:

1. Vulnerability is reported privately
2. Maintainers confirm and develop a fix
3. Fix is released
4. CVE is requested (if applicable)
5. Public disclosure after users have had time to update

Thank you for helping keep spline-lstm secure!
